# Skill: change-design

## 目的

根据实现反馈或需求变更修改已有技术设计。

## 触发条件

- 用户要求修改设计
- 实现过程中暴露问题
- 发现新约束

## 输入

- 现有 design.md 路径
- 变更说明
- 变更原因

## 流程

### 步骤 1：读取现有设计

加载 `.cantrip/workspace/{task-id}/design.md`

### 步骤 2：评估影响

- 哪些模块受影响
- 哪些任务需要修订
- 对时间线的影响

### 步骤 3：应用变更

更新：
- 架构（如需要）
- 数据结构
- 接口
- 实现计划

### 步骤 4：更新风险评估

根据变更重新评估风险。

## 输出

更新后的 `.cantrip/workspace/{task-id}/design.md`（含变更历史等）。

## 输出格式

```
📐 设计已更新

**变更**:
- 架构: {变更或「无」}
- 数据结构: {变更或「无」}
- 受影响任务: {n}

**新增风险**:
- {如有}

**对实现的影响**:
- 需重做任务: {list}
- 需新增任务: {list}

请审阅变更。
```
