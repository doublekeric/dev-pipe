description: 开发任务入口。分析任务类型，初始化项目（如需要），路由到对应工作流。
---

分析任务：$ARGUMENTS

执行流程：

1. **检查初始化**
   - 检查 `.dev-pipe/` 目录是否存在
   - 如不存在，先执行初始化流程

2. **分析任务类型**
   根据 $ARGUMENTS 中的关键词判断：
   - "实现/开发/添加/新建" → 功能开发流程
   - "修复/解决/bug/问题" → Bug 修复流程
   - "继续/恢复" → 读取状态继续

3. **执行对应流程**
   - 功能开发：需求分析 → 方案设计 → 代码实现 → 验证
   - Bug 修复：问题分析 → 定位根因 → 实施修复 → 验证

4. **状态持久化**
   - 更新 `.dev-pipe/workspace/` 中的任务状态
   - 任务进度可随时恢复
